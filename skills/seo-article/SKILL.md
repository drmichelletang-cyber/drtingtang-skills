# SEO 文章寫作系統

叮噹醫師品牌專屬的 SEO 長文撰寫流程。融合 21 天 SEO 課程三週框架與 AIOSEO 技術優化標準。

使用此技能前，先讀取 `/drtingtang` 和 `/9s-course` 技能以取得品牌語調與課程內容。

---

## 撰寫前：必須先與使用者確認三件事

開始寫文章之前，必須完成以下確認流程。使用 AskUserQuestion 工具，以「選擇題」形式呈現，讓使用者快速決策。

### 確認 1：黃金關鍵字 + 次要關鍵字

先用 WebSearch 研究該主題的搜尋熱度與競爭狀況，然後提供 3–4 組關鍵字組合讓使用者選擇。每組包含 1 個主關鍵字 + 1–2 個次要關鍵字。

所有關鍵字組合必須通過 3S 過濾器：
- **Searchable**：確認有搜尋量（用 Google 搜尋建議驗證）
- **Suitable**：叮噹醫師能結合高齡醫學 + 東方哲學提供獨特觀點
- **Specific**：一篇 2,000–3,500 字的文章寫得完

範例選項格式：
```
選項 A：主關鍵字「40歲後健康管理」＋ 次要「中年養生方法」「中年健康規劃」
選項 B：主關鍵字「中年健康檢查項目」＋ 次要「40歲健檢推薦」「健檢頻率」
選項 C：主關鍵字「中年體力下降怎麼辦」＋ 次要「40歲體能訓練」「肌力流失」
```

### 確認 2：搜尋意圖

根據選定的關鍵字，判斷最可能的搜尋意圖，提供選項讓使用者確認：

- **資訊型（Informational）**：使用者想了解知識 → 寫教學文/指南文
- **商業型（Commercial）**：使用者在比較選項 → 寫比較文/推薦文
- **交易型（Transactional）**：使用者準備行動 → 寫銷售頁/Landing Page

### 確認 3：Google 搜尋建議（User Queries）

用 WebSearch 蒐集該關鍵字相關的 People Also Ask、自動完成建議、相關搜尋，整理成 8–12 個使用者常問問題，讓使用者勾選要納入文章的項目。

範例格式：
```
□ 40歲後該做哪些健康檢查？
□ 中年體力下降是正常的嗎？
□ 40歲開始運動會不會太晚？
□ 中年飲食要怎麼調整？
☑ 由使用者勾選要寫進文章的問題
```

使用者選定的問題將轉換為文章的 H2 標題。

**三件事都確認完畢後，才開始撰寫。**

## 第一階段：文章骨架（Week 1 框架）

### 標題公式

根據確認的搜尋意圖，套用對應公式：

- **資訊型**：「[關鍵字]是什麼？[權威身份]用[數字]個步驟帶你搞懂」
- **商業型**：「[選項A] vs [選項B]｜[權威身份]的[年數]年臨床觀察」
- **交易型**：「[痛點]？[方案名稱]幫你[具體結果]」

技術要求：
- 50–60 字元以內（避免 Google 截斷）
- 主關鍵字放在標題前半段
- 包含明確利益點或數字

### Meta 描述

- 120–155 字元
- 包含主關鍵字
- 有行動呼籲或價值預告

### 前言公式：痛點 → 同理 → 預告

1. **痛點**（1–2 句）：描述讀者正在經歷的具體困擾
2. **同理**（1–2 句）：讓讀者感覺「你懂我」
3. **預告**（1 句）：告訴讀者這篇文章會給他什麼

前言必須在前 100 字內自然出現主關鍵字。

### H2 結構

將使用者選定的 User Queries 轉換為 SEO 友善的 H2：
- 每個 H2 直接回應一個使用者問題
- H2 中自然包含主關鍵字或語意相關詞
- 排列順序由淺入深、由問題到解方

## 第二階段：結構化積木（Week 2 框架）

核心原則：AI 不需要優美的文筆，它需要清晰的數據標記。

### 12 種積木類型

| 積木 | 用途 | 呈現方式 |
|:---|:---|:---|
| 流程引導型（How-to） | 編號步驟 | 有序列表 |
| 決策輔助型（Choice） | 比較選項 | 比較表格 |
| 觀念矯正型（Pitfalls） | 正確 vs 錯誤 | 雙欄對比框 ✅ vs ❌ |
| 資訊強化型（Info/Note/Warning） | 補充重要資訊 | 引用區塊或 callout |
| 案例展示型（Success/Case/Example） | 真實案例 | 案例卡片 |
| 頭尾呼應型（Hook + Action） | 開頭吸引 + 結尾 CTA | Hook Box + Action Checklist |

### 自動插入規則

每 300 字自動插入一個結構化元素。AI 根據上下文判斷最適合的積木類型並直接生成。判斷依據：

- 正在講步驟/流程 → 流程引導型（編號步驟）
- 正在比較兩個以上選項 → 決策輔助型（比較表格）
- 正在澄清迷思或常見錯誤 → 觀念矯正型（對比框）
- 正在補充數據或研究 → 資訊強化型（引用區塊）
- 正在舉例或說故事 → 案例展示型（案例卡片）

**若上下文無法明確判斷該放哪種積木，暫停撰寫並詢問使用者。**

### 可讀性規範

- 段落最多 3–4 句
- 句子盡量控制在 20 字以內
- 避免連續超過 4 句的純文字段落
- 避免被動語態，以主動語態為主
- 使用轉折詞連接段落（然而、因此、換句話說、更重要的是）
- 避免連續句型重複（不要每句都同一個開頭）

## 第三階段：E-E-A-T 強化（Week 3 框架）

核心原則：AI 可以寫出正確的廢話，但寫不出信任感。

### Experience（經驗）
- 加入叮噹醫師第一人稱臨床觀察
- 使用「在我二十年的行醫經驗中⋯」「我在英國執業時⋯」等框架
- 具體場景 > 抽象論述

### Expertise（專業）
- 引用至少 1 位領域專家的觀點或研究
- 格式：「根據 [專家姓名]，[機構名稱] 的研究指出⋯」
- 資料來源必須是權威機構（學術期刊、WHO、國健署等），不引用競品部落格

### Authoritativeness（權威）
- 引用具體數據並標明出處
- 格式：「根據 [機構名稱]（[年份]）的數據，[具體數字]⋯」
- 優先來源：Nature、Lancet、NEJM、WHO、台灣國健署

### Trustworthiness（可信度）
- 不做結果保證，承諾過程品質
- 不使用「讓好習慣自動發生」等無法兌現的語言

### FAQ 精選摘要區

文章結尾加入 3–5 題 FAQ：
- 每個答案 50–75 字
- 第一句直接回答問題
- 問題用 H3 標記
- 可標記 FAQ Schema（JSON-LD）

### 作者框

文末固定加入：

```
## 關於作者

**叮噹醫師 鄭丁靚**（Dr. TingTang）
MBBCh, MRCP UK｜高齡醫學科醫師

幫助 35–55 歲族群在健康斷崖到來之前，完成人生後半場的作戰部署。

👉 了解更多：drtingtang.com
```

## AIOSEO 技術檢查清單

每篇文章完稿後，逐項自我檢查（對齊 TruSEO 70+ 分標準）：

### 基礎 SEO
- 主關鍵字出現在：標題、H1、URL、Meta 描述、前 100 字、至少一個 H2
- 關鍵字密度約 1%（自然分佈不堆砌）
- 次要關鍵字與語意相關詞散佈全文
- URL 簡短乾淨，包含主關鍵字
- 一頁只有一個 H1

### Meta 標籤
- Meta Title：50–60 字元，主關鍵字在前半段
- Meta Description：120–155 字元，含主關鍵字與行動呼籲

### 連結策略
- 至少 2–3 個內部連結（連到相關文章或課程頁）
- 至少 1–2 個外部連結（連到權威資料來源）
- 錨文字具描述性（不用「點這裡」）

### E-E-A-T 驗證
- 至少 1 段第一人稱臨床經驗描述
- 至少 1 個專家引述或研究引用
- 至少 1 個具體數據（附出處）
- 文末有作者框
- FAQ 區 3–5 題，每題 50–75 字

### Schema 結構化數據（進階）
- Article Schema（文章類型、作者、發布日期）
- FAQ Schema（JSON-LD）
- Author Schema（資格與關聯網站）

## 品牌語調提醒

- **人設**：長壽策略長（嚮導與設計者）
- **語調**：溫暖權威——像你信任的醫師朋友在講重要的事
- 用「帶你」「幫你設計」，不用「陪你」「一起」
- 東方哲學自然融入，每篇 1–2 句，先講數據再用智慧收束
- 承諾過程品質，不承諾結果保證

## 完整工作流程

```
[撰寫前確認（選擇題）]
  1. 黃金關鍵字 + 次要關鍵字 → 3–4 組選項
  2. 搜尋意圖 → 資訊型/商業型/交易型
  3. User Queries → 勾選要納入的問題

[撰寫]
  4. 標題公式 + Meta 描述
  5. 前言（痛點→同理→預告）
  6. H2 骨架（從選定的 Queries 轉換）
  7. 正文 + 結構化積木（每 300 字自動插入，不確定就問）
  8. E-E-A-T 注入
  9. FAQ 精選摘要（3–5 題）
  10. 作者框

[完稿檢查]
  11. AIOSEO 技術檢查清單
  12. 全文健檢（結構 + E-E-A-T + 可讀性）
```
